# PRD：新闻信号显示优化

## 背景

前端新闻动态页面（News）中，每个币种标签旁显示一个数字，如 `BTC ▲ 960`。用户反馈不清楚该数字的含义，且数值普遍超过 100，与直觉中的"评分/分数"认知不符。

经排查，该数字来自后端接口返回的 `weighted_score` 字段，实际含义是**该币种在指定时间窗口内所有新闻加权分数的累加和**，而非单条新闻的评分。

此外，用户希望将静态数字显示改为**图表化展示**，方便观察信号随时间的变化趋势。

### 当前问题

| 问题 | 说明 |
|------|------|
| 语义不清 | 字段名 `weighted_score` 暗示是"评分"，实际是"累计和" |
| 数值偏大 | 12条新闻 × 每条 ~80分 = 960，用户无法直观理解 |
| 缺少上下文 | 未展示新闻条数，无法判断数字是"强信号"还是"多条叠加" |
| 无法观察趋势 | 静态数字无法展示信号随时间的变化 |
| 缺少方向语义 | 仅展示 `▲` 上涨符号，下跌和中性信号未定义 |

---

## 目标

1. 让用户**一眼看懂**数字代表什么
2. 数值范围回到**直观可比较**的量级（0-100 的平均信号强度）
3. 保留**新闻条数**信息，不丢失上下文
4. **图表化展示**信号趋势，方便观察时间维度的变化
5. 提供信号**可靠性参考**，帮助用户判断信号置信度

---

## 术语定义

| 术语 | 定义 |
|------|------|
| 信号强度（Signal Strength） | 单条新闻的 `weighted_score`，范围 0-100，由 LLM 标注 |
| 平均信号强度（Avg Signal Strength） | `weighted_score / event_count`，范围 0-100，反映该币种新闻情绪的平均水平 |
| 累计信号（Cumulative Signal） | 原 `weighted_score`，时间窗口内所有新闻信号强度的累加和 |
| 信号方向（Direction） | bullish / bearish / neutral，由后端根据平均信号强度及情绪分析结果判定 |
| 置信度（Confidence） | 基于新闻条数的信号可靠性指标：≥10条=高，5-9条=中，1-4条=低 |

---

## 设计方案

**一句话概括**：用"平均信号强度 + 方向图标 + 新闻条数"替代原始累计值，并用折线趋势图替换静态卡片。

### 数字显示

主显示值为后端计算的平均信号强度（0-100），辅以信号方向图标和新闻条数：

```text
BTC  ▲ 80 (12)        ← 上涨，平均强度 80，12条新闻
ETH  ▼ 45 (8)         ← 下跌，平均强度 45，8条新闻
SOL  ● 62 (3)         ← 中性，平均强度 62，3条新闻
```

- 方向：`▲` 上涨（绿色）/ `▼` 下跌（红色）/ `●` 中性（灰色），由后端 `direction` 字段驱动
- 条数：括号内显示新闻条数，Hover 时展示置信度

### 图表可视化

用**折线趋势图 + 热力色带**替换现有 SignalChips 卡片：

- **折线**：每个币种一条，展示平均信号强度随时间的变化趋势，Y 轴 0-100
- **热力色带**：背景层以颜色深浅映射新闻条数密度，不与折线争夺视觉焦点
- **默认 Top 5**：按新闻条数降序自动选取最活跃的 5 个币种
- **时间粒度**：支持 hourly（回看 48h）和 daily（回看 30d）切换
- **图例交互**：点击图例可显示/隐藏单个币种；Hover 显示详情（时间、分数、条数、置信度）

## 接口规格

### 扩展现有新闻信号接口

新增字段（`event_count = 0` 时均返回 `null`）：

```json
{
  "symbol": "BTC",
  "weighted_score": 960,
  "event_count": 12,
  "avg_weighted_score": 80,
  "direction": "bullish",
  "confidence": "high"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `avg_weighted_score` | `float \| null` | 平均信号强度（后端计算） |
| `direction` | `enum \| null` | `bullish` / `bearish` / `neutral` |
| `confidence` | `string \| null` | `high`（≥10条）/ `medium`（5-9条）/ `low`（1-4条） |

### 新增趋势数据 API

**Endpoint**: `GET /api/news/signals/trend`

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `granularity` | `string` | `daily` | `hourly` 或 `daily` |
| `days` | `int` | `30` | 回看天数（hourly 上限 2，daily 上限 90） |
| `symbols` | `string` | — | 逗号分隔，为空则返回 Top 5（按 event_count） |
| `limit` | `int` | `5` | 返回币种数量上限 |

**响应**:

```json
{
  "granularity": "daily",
  "time_range": { "start": "2026-04-15", "end": "2026-05-15" },
  "symbols": [
    {
      "symbol": "BTC",
      "direction": "bullish",
      "avg_weighted_score": 80,
      "event_count": 12,
      "confidence": "high",
      "trend": [
        { "time": "2026-05-13", "avg_score": 75, "event_count": 8, "direction": "bullish" },
        { "time": "2026-05-14", "avg_score": 82, "event_count": 12, "direction": "bullish" },
        { "time": "2026-05-15", "avg_score": 80, "event_count": 5, "direction": "neutral" }
      ]
    }
  ]
}
```

**数据来源**: 从 `news_analysis` 表按 `granularity` 实时聚合（daily 30d × 5 币种 ≈ 150 行，查询 < 200ms）。数据量增大时再引入预计算缓存。
